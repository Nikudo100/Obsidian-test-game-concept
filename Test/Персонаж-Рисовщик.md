# Структура умений персонажа

## Особенность

**Чернильные печати**

- позволяет рисовать символы
    
- создаёт заклинания
    

Это **ядро персонажа**.

---

## Атака

**Чернильное перо**

Дальний авто-выстрел.

Эффект:

- выпускает капли чернил
    
- небольшой DoT
    

Это нужно чтобы персонаж **не был беспомощным**, когда не рисует.

---

## Мощь

**Быстрый символ**

Мгновенно активирует **последний нарисованный символ**.

Пример:

Игрок ранее нарисовал

~

Нажал мощь → волна.

Это ускоряет геймплей.

---

## Особая атака

**Чернильный всплеск**

AOE вокруг героя.

Эффект:

- замедление
    
- наносит урон
    

Используется чтобы **защититься во время рисования**.

---

## Защита

**Чернильная завеса**

Короткий щит.

Дополнительно:

если активировать во время рисования:

рисунок нельзя прервать

---

## Рывок

**Чернильный след**

Рывок оставляет:

лужу чернил

которая замедляет врагов.

---

## Ульта

### Основная

**Живое полотно**

Игрок быстро рисует **3 символа**.

Они автоматически кастуются.

Пример:

↑  
~  
O

→ стрела  
→ волна  
→ щит

---

### Альтернативная

**Чернильный демон**

Большой призыв.

Эффект зависит от **последнего символа**.

Например:

Z → электрический демон  
O → защитный демон



Особенность 
при нажатии Q открывается белый квадрат (холст) на котором можно рисовать

# Почему именно такие символы

Они отличаются по:

- количеству линий
    
- направлению
    
- кривизне
    

| символ | тип         |
| ------ | ----------- |
| ↑      | направление |
| ~      | кривая      |
| T      | структура   |
| O      | замкнутый   |
| Z      | ломанный    |
| V      | угол        |
| X      | крест       |
| —      | линия       |
| \|     | линия       |
| @      | спираль     |

Алгоритм их **легко различает**.

# Комбинации (2 символа)

Комбинация — это **быстрое рисование двух символов подряд**.

Например:

↑ + ~

---

# Как система будет распознавать комбинации

Алгоритм простой.

Игрок рисует:

stroke1  
stroke2

Система:

1. распознаёт первый символ
    
2. распознаёт второй символ
    
3. проверяет комбинацию
    

symbolA + symbolB

---

# Важное правило

Комбинации **НЕ симметричны**.

↑ + Z ≠ Z + ↑

Это увеличивает количество заклинаний.

---

# Сколько всего получится

10 символов.

Комбинации по 2:

10 × 9 = 90

Но реально в игре лучше оставить:

**15–20 комбинаций.**

---

# Как сделать систему ещё круче

Можно добавить **типы символов**.

|тип|символ|
|---|---|
|атака|↑ Z —|
|контроль|~ @|
|защита|O|
|призыв|V|
|взрыв|X|

Тогда комбинации становятся **логичными**.

Например

Attack + Control

=

атака с эффектом контроля.

---

# Очень важный геймдизайн момент

Игрок должен **помнить символы интуитивно**.

Например:

O = защита  
Z = молния  
~ = вода  
↑ = выстрел





вместо провала:

Weak Ink Shot (слабая струйка чернил)


# 1. UI рисования (идеальный для боевой игры)

Главная цель:  
**игрок должен нарисовать символ за 0.3–0.7 сек и сразу вернуться в бой.**

Поэтому **большой холст не нужен**.

Лучше сделать **маленький круг вокруг героя**.

### Когда нажимаем Q

Вокруг героя появляется **магический круг**.

Пример:

        Z  
     ~     ↑  
  
        O

В центре — герой.

Это одновременно:

- **холст**
    
- **подсказка символов**
